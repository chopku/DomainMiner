# DomainMiner

**DomainMiner** (ранее DNSMapper) — это легковесная консольная утилита на Python для автоматического разрешения (DNS-lookup) списков доменных имен в IP-адреса и генерации файлов статических маршрутов (для роутеров Keenetic, Cisco и др.).

## Возможности

* **Парсинг файлов конфигурации:** Читает домены из текстовых файлов в заданной директории.
* **Игнорирование комментариев:** Поддерживает пустые строки и комментарии, начинающиеся с `#` (включая инлайновые комментарии в конце строки).
* **DNS-разрешение IPv4:** Извлекает только валидные IPv4-адреса для каждого домена с настраиваемым таймаутом.
* **Правильная сортировка IP:** Сортирует сгенерированные IP-маршруты по числовым значениям (используя библиотеку `ipaddress`), а не лексикографически.
* **Форматирование маршрутов:** Записывает результаты в формате:
  `ip route <IP> 255.255.255.255 <domain_name>`
* **Исключение дубликатов:** Защищает итоговые файлы от повторяющихся IP-адресов.

## Требования

* Python 3.9 или выше.
* Не требует установки сторонних библиотек (использует только стандартные библиотеки Python: `argparse`, `ipaddress`, `socket`, `pathlib`).

## Установка и подготовка

1. Клонируйте репозиторий или скачайте исходный код:
   ```bash
   git clone https://github.com/chopku/DomainMiner.git
   cd DomainMiner
   ```
2. Создайте в корне проекта папку `domain_list` и поместите туда текстовые файлы со списками доменов (например, `gemini.txt` или `openai.txt`). Каждый файл будет обработан как отдельный сетевой интерфейс.
   
   **Пример содержимого файла `domain_list/gemini.txt`:**
   ```text
   # Домены для Gemini
   gemini.google.com
   aistudio.google.com # API студия
   ```

## Использование

Для запуска разрешения доменов выполните команду:
```bash
python resolver.py
```

### Параметры командной строки

Вы можете гибко настраивать работу утилиты с помощью флагов:

| Флаг | Описание | Значение по умолчанию |
| --- | --- | --- |
| `-d`, `--dir` | Директория с исходными текстовыми файлами доменов | `domain_list` |
| `-o`, `--output-dir` | Директория для сохранения готовых маршрутов | `resolved` |
| `-t`, `--timeout` | Таймаут DNS-запроса для каждого домена (в секундах) | `5.0` |
| `-v`, `--verbose` | Детальный вывод логов уровня DEBUG | Отключен (уровень INFO) |

**Пример команды с параметрами:**
```bash
python resolver.py --dir custom_domains --output-dir my_routes --timeout 3.0 --verbose
```

## Структура проекта

* `resolver.py` — основной исполняемый файл утилиты.
* `domain_list/` — папка со списками доменов (входные данные).
* `resolved/` — папка со сгенерированными статическими маршрутами (выходные данные).
* `.gitignore` — исключение временных и сгенерированных файлов из системы контроля версий.
